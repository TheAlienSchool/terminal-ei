  2. Keep the experience light.
  3. Build local-first.
  4. Use “::” as direction.

Alt-Text in Terminal E.I. :: Ground Crew Guide

Alt-text in this terminal is a teaching layer.

It serves three purposes:

  1. Describe the element clearly.
  2. Offer one simple insight about awareness or navigation.
  3. Stay readable to someone visiting for the first time.

When you write alt-text here:

  1. Stay plain :: use everyday language, no jargon.
  2. Name the object :: say what it is.
  3. Share one truth :: what this element helps the traveler notice or do.
  4. Keep it self-contained :: each line should make sense on its own.
  5. Align with Ping :: Echo :: Direction when possible.
  6. Use .visually-hidden for teaching lines near the element.

If you follow these guidelines, the alt-text will act like a quiet guide for anyone using a screen reader and a hidden decoder ring for anyone who inspects the code.
